# Security Policy

## Scope

This repository is a reference package for task-contract design patterns and example skill structures.

It is not a full security framework, and its presence does not replace:
- authentication
- authorization
- sandboxing
- secret management
- audit logging
- runtime isolation

## Reporting a concern

If you find an issue in:
- schema semantics
- approval boundary examples
- risky example phrasing
- adapter behavior that appears unsafe

please open a private report with enough context to reproduce the concern.

## Security posture

This project focuses on improving safety through structure:
- explicit scope
- explicit assumptions
- explicit approval requirements
- structured handoff
- structured downstream results

It does not claim to eliminate model or runtime risk on its own.
